4.1. GRUB

To boot Debian GNU/kFreeBSD via an existing GRUB installation, add the following to your /boot/grub/menu.lst file:

title Debian GNU/kFreeBSD
root (device)
kernel /boot/loader

The device you'll need to specify depends on which partition and slice your system has been installed to.
If you chose to install to the first partition, inside the first slice on your system's first disk (ie. /dev/ad0s1a), you'd specify (hd0,0,a) in your GRUB configuration.

4.2. X.Org

X Server configuration is performed in the typical Debian fashion, via:

dpkg-reconfigure xserver-xorg

Mouse device node names are the only major difference involved in X configuration compared to a Debian GNU/Linux system. Your mouse will use one of the following device names:

/dev/ums*        - USB
/dev/psm*        - PS/2
/dev/mse*        - Bus mouse
/dev/cuaa*       - Serial
/dev/sysmouse    - Mouse daemon (moused)

For example, the first USB mouse detected by your system would be named /dev/ums0, the second /dev/ums1, and so on.

The moused package provides a console mouse daemon similar to the gpm package found on Debian GNU/Linux. If you would like to have mouse functionality under both X11 and regular virtual terminals, install moused, configure it to use your particular mouse type via /etc/moused.conf, restart the daemon, then specify /dev/sysmouse as your mouse device in /etc/X11/xorg.conf, along with "auto" for the protocol type.

4.3. Misc.

apt-get key check - To stop apt from complaining about key checks, import the debian-ports.org mirror GPG key with the following command:

gpg --keyserver subkeys.pgp.net --no-default-keyring --primary-keyring /etc/apt/trusted.gpg --recv 0917E535

Keyboard map - To configure your keyboard mapping, install the kbdcontrol package and select a layout from the displayed list.

PF - Debian GNU/kFreeBSD includes the OpenBSD Packet Filter by default, with userland tools such as pfctl included in the freebsd-utils package. A separate PF-specific package will be prepared in the future.

Development - When building packages make sure you have the latest autotools-dev package installed. Many packages have outdated config.* files. See PORTING for more information.
