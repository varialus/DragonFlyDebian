Supported hardware

Debian GNU/kFreeBSD runs on the i386 architecture ("IA32", "x86") and supports any hardware that FreeBSD does. See "Supported hardware" in the FreeBSD handbook for details.

Installation media

There are currently two methods of installation; using a FreeBSD install CD or using a GNU/kFreeBSD live CD.
The live CD contains a minimal environment for bootstrapping a Debian GNU/kFreeBSD system. One thing the live CD is not currently capable of, is partitioning. To create the required partitions/slices, you will need a working Linux or BSD system. This is why it is recommended that you install using an official FreeBSD installation CD, such as the 5.x mini install iso.

If you choose to install with the FreeBSD CD you will still need a copy of the GNU/kFreeBSD live CD. You can find out more in the "Installing with a FreeBSD CD" section. Otherwise, follow "Installing with the live CD".

Regarding the live CD and partitioning, whatever software you use to do this, the important thing is that it has to support creating a FreeBSD partition and a root and swap slice. Something like GNU Parted is capable of this.

Disk space

If you're planning to use your GNU/kFreeBSD system for development purposes (building packages etc.), around 2Gb of hard disk space is a good idea. If you're just giving Debian GNU/kFreeBSD a try, you could probably get away with around 500mb.
